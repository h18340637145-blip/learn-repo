# NodePath 完整 Node.js 课程体系设计

## 1. 背景

NodePath 当前已经验证了“阅读概念、观察案例、预测结果、答对后运行、查看可视化与日志、阅读总结”的学习模式。现有内容只有 3 个知识点和 1 个阶段项目，无法支撑系统学习，也无法体现 Node.js 官方学习资料的完整知识结构。

本设计以 [Node.js 官方 Learn](https://nodejs.org/en/learn) 为主要内容依据，将官方主题按学习依赖重新编排，而不是照搬官方导航顺序。课程面向已经掌握 JavaScript 基础语法、希望系统学习 Node.js 后端开发的学习者。

## 2. 已确认的产品决策

- 课程采用“实用必修主线 + 高级选修轨道”，不把 Node-API 等底层主题设为毕业前置条件。
- 必修主线包含 10 个阶段、80 个知识点、10 个阶段项目和 1 个最终综合项目。
- 每个知识点至少包含 3 道题目变体，基础题库规模为 240 道；根据主题复杂度可扩展到 320 道。
- 普通知识点使用经过验证的预设运行轨迹；阶段项目和综合项目通过隔离沙箱真实执行。
- 课程以 ESM 为模块体系主线，CommonJS 作为完整对照内容，并保留 `require` 缓存案例。
- 最终综合项目为“实时协作任务平台”。
- 首期使用浏览器本地存储保存学习进度，但通过存储接口隔离具体实现，为后续接入 Supabase 保留边界。

## 3. 范围

### 3.1 本轮范围

- 建立完整课程目录、课程规格和官方来源追踪机制。
- 将课程数据从 `app/learning-studio.tsx` 的 UI 实现中解耦。
- 支持 10 类可视化表现和统一运行事件格式。
- 支持错误答案的定向反馈、正确答案解释、错题记录和复习队列。
- 支持课程进度、阶段解锁、阶段项目和最终项目解锁。
- 为受控项目模板提供真实的隔离执行能力。
- 建立内容、逻辑、集成和端到端四层验证体系。

### 3.2 明确不在本轮范围

- 任意代码编辑、任意源代码上传和任意 npm 依赖安装。
- 教师端可视化课程编辑器。
- 排行榜、社交关系、证书和付费系统。
- Node-API、WebAssembly 等选修主题的首期完整课程制作。
- 首期账号系统和云端进度同步。

## 4. 课程地图

必修主线固定为 10 个阶段，每阶段 8 个知识点和 1 个项目，共 80 个知识点和 10 个阶段项目。

### 阶段 01：运行时与命令行

知识点：Node.js 运行时、Node.js 与浏览器的差异、V8 与运行过程、版本与 LTS 概念、执行脚本、REPL、`process` 与命令行参数、环境变量与终端输出。

阶段项目：CLI 系统信息探测器。

### 阶段 02：模块、包与 TypeScript

知识点：ESM 导入导出、模块解析、`package.json` 与 `type`、内置模块的 `node:` 前缀、npm 与依赖类型、语义化版本和 npm scripts、CommonJS 与 `require` 缓存、Node.js 中的 TypeScript 使用方式。

阶段项目：依赖与配置检查器。

### 阶段 03：异步运行时与事件

知识点：回调模式、Promise、`async/await`、异步错误传播、事件循环阶段、微任务与 `process.nextTick`、`setImmediate` 与计时器、`EventEmitter` 与任务取消。

阶段项目：并发任务调度器。

### 阶段 04：文件、Buffer 与 Stream

知识点：`path` 与文件 URL、`fs/promises`、目录和文件元数据、文件监听、Buffer 与字符编码、Readable Stream、Writable 与 Transform Stream、`pipe` 与背压。

阶段项目：CLI 日志分析器。

### 阶段 05：HTTP 基础

知识点：HTTP 事务生命周期、创建 Server、请求对象、响应对象、Header 与状态码、路由和查询参数、请求体与流式解析、流式响应与 Fetch。

阶段项目：流式静态文件服务器。

### 阶段 06：API 与服务设计

知识点：REST 资源建模、输入验证、统一错误模型、配置边界、结构化日志、超时控制、`AbortSignal`、健康检查与优雅关闭。

阶段项目：任务管理 REST API。

### 阶段 07：进程与并发

知识点：阻塞事件循环、libuv 线程池、`child_process`、`worker_threads`、进程间消息传递、共享内存边界、Cluster、并发模型选择。

阶段项目：Worker Pool 报表生成器。

### 阶段 08：实时通信

知识点：轮询与长轮询、SSE、WebSocket 握手、连接生命周期、心跳与超时、消息广播、实时流量背压、断线恢复与幂等处理。

阶段项目：实时任务通知服务。

### 阶段 09：测试与安全

知识点：`node:test`、断言、测试生命周期、Mock、代码覆盖率、集成测试、权限与密钥边界、依赖和 Web 安全实践。

阶段项目：经过测试的鉴权服务。

### 阶段 10：诊断与生产工程

知识点：Inspector 调试、CPU Profiling、Heap Snapshot、GC 追踪、火焰图、性能基线、生产配置与可观测性、发布与故障检查流程。

阶段项目：生产故障诊断实验室。

### 最终综合项目

“实时协作任务平台”复用全部阶段能力：模块边界、REST API、实时同步、Worker 任务、鉴权安全、自动化测试、性能诊断和生产运行。最终项目采用受控代码补丁和真实沙箱执行，每个里程碑均给出可视化项目结果与知识总结。

### 选修轨道

选修内容包括 Node-API 原生扩展、WebAssembly、HTTP/2 与 TLS 深入、Linux perf 和高级内存诊断。选修内容使用与必修课程相同的课程规格，但不参与主线解锁和毕业判定。

## 5. 官方内容覆盖与版本治理

官方 Learn 顶层主题映射如下：

| 官方主题 | NodePath 阶段 |
| --- | --- |
| Getting Started、Command Line | 01 |
| Package Management、TypeScript | 02 |
| Asynchronous Work | 03 |
| File System | 04 |
| HTTP | 05、06 |
| Concurrency | 07 |
| WebSocket 及实时扩展 | 08 |
| Test Runner、Security | 09 |
| Diagnostics、生产实践 | 10 |

REST、SSE、鉴权和生产可观测性属于实用工程扩展，用于把官方基础知识连接成可交付的后端能力。课程元数据必须区分“官方直接主题”和“工程扩展主题”。

每个知识点必须记录：

- 官方来源标题和 URL。
- 来源类型：官方直接主题或工程扩展主题。
- 适用的 Node.js 版本范围。
- 最后核验日期。
- 学习目标和先修知识。
- 示例代码的期望日志或结构化结果。

内容校验需要生成覆盖矩阵。所有必修官方主题必须至少映射到一个知识点；链接失效、版本过期或长期未核验时，持续集成应给出明确告警。

## 6. 单课学习闭环

每个知识点使用统一的七步流程：

```text
概念与记忆钩子
  -> 阅读可运行案例
  -> 预测运行结果
  -> 错误时获得定向提示并重试
  -> 正确后执行案例
  -> 观察可视化过程和终端日志
  -> 阅读总结并进入复习队列
```

首次答错只展示与该错误选项对应的误区提示，不直接暴露正确答案。第二次答错可展示一个关键运行步骤。答对后必须解释正确选项成立的原因，以及其他选项不成立的原因。

当前实现中所有课程共用的 Event Loop 错误提示必须移除，改为每个选项独立配置反馈。

## 7. 题库模型

每个知识点至少包含三类题目：

1. 预测题：判断代码输出、执行顺序或运行状态。
2. 诊断题：定位错误原因、性能问题或不安全写法。
3. 迁移题：在不同代码场景中应用同一规则。

预测题是首次学习的主挑战；诊断题和迁移题主要进入复习队列。阶段项目仍采用“项目说明、待补全代码、选择受控补丁、真实运行、视觉结果、知识总结”的交互模式。

课程规格至少包含以下字段组：

- 标识：课程 ID、阶段 ID、顺序、难度、预计时长。
- 教学：学习目标、先修知识、概念讲解、关键点、记忆钩子。
- 案例：文件集合、入口文件、展示代码、支持的 Node.js 版本。
- 题目：题型、题干、选项、答案、各选项反馈、正确答案解释。
- 执行：预设轨迹或沙箱模式、期望输出、超时策略。
- 可视化：可视化类型、通道、帧或事件映射。
- 总结：知识规则、常见误区、复习提示。
- 来源：官方 URL、来源类型、版本范围、核验日期。

## 8. 内容与组件边界

课程内容不再定义在 `app/learning-studio.tsx` 内。目标边界如下：

```text
课程索引
  -> 阶段清单
    -> 课程规格与题库
      -> 执行适配器
        -> 可视化注册表
          -> 通用学习界面
```

- 课程索引负责阶段顺序、先修关系和解锁条件。
- 课程规格只描述内容，不包含 React 状态和页面分支。
- 执行适配器统一预设轨迹和真实沙箱输出。
- 可视化注册表根据事件类型选择模块图、事件队列、Stream、HTTP、Worker、测试、性能或应用预览组件。
- 通用学习界面只消费课程规格和统一执行事件，新增课程不需要修改页面组件。

## 9. 混合执行架构

### 9.1 预设轨迹

普通知识点使用作者编写并经过验证的结构化运行轨迹。轨迹需要同时驱动视觉状态、终端日志和教学注释。它不依赖网络或服务端执行，确保播放速度和教学表达稳定。

### 9.2 真实沙箱

阶段项目和综合项目通过服务端执行接口进入独立沙箱。浏览器仅提交课程 ID、答案 ID 和尝试 ID。服务端根据课程 ID 加载可信项目模板，再应用答案对应的受控代码补丁；浏览器不能上传任意源代码。

真实执行链路为：

```text
请求校验
  -> 加载可信模板和受控补丁
  -> 创建隔离任务
  -> 沙箱运行 Node.js
  -> 标准化运行事件
  -> 通过 SSE 传输
  -> 前端可视化与终端输出
```

Next.js 应用进程不得直接执行课程代码。沙箱默认禁止外部网络，只允许课程需要的内部回环通信；使用临时工作目录和只读运行时，并限制 CPU、内存、执行时长、进程数和输出大小。依赖必须预安装且使用白名单，运行期间不得执行 `npm install`。宿主密钥和环境变量不能暴露给沙箱。

### 9.3 统一事件

预设轨迹和真实沙箱统一输出以下语义：时间、阶段、事件类型、可视化通道、状态数据、日志和教学注释。前端不根据来源编写两套学习界面。

## 10. 错误处理与降级

- 错误答案不触发真实执行，只进入定向教学反馈。
- 课程代码的语法错误和运行错误作为学习事件展示，并定位到对应文件和代码位置。
- 超时、内存超限、进程超限和输出超限必须分别显示原因。
- 沙箱服务不可用时，可回退到该项目预先验证的运行轨迹，并明确标记“模拟运行”。
- 学习者切换课程或取消运行时，服务端终止任务，客户端丢弃迟到事件。
- 课程内容无法加载或规格校验失败时，页面显示课程 ID 和可恢复操作，不渲染半完整课程。

## 11. 学习进度与解锁

知识点状态包括：未解锁、可学习、学习中、已完成和待复习。学习者答对主预测题、完整观察运行结果并到达总结后，知识点记为已完成。

完成当前阶段全部必修知识点后解锁阶段项目；完成阶段项目后解锁下一阶段。10 个阶段项目全部完成后解锁最终综合项目。复习题和错题队列用于巩固，不阻塞后续阶段。

首期进度通过独立的进度仓储接口写入浏览器本地存储。React 组件不能直接散落调用 `localStorage`。后续接入 Supabase 时，只替换仓储实现，并增加登录后的本地进度合并策略。

## 12. 可视化类型

首期可视化注册表支持以下 10 类表现：

1. 模块依赖与缓存图。
2. 调用栈、微任务和事件循环队列。
3. Buffer、Readable、Writable 和背压状态。
4. HTTP 请求响应生命周期。
5. 进程、线程池和 Worker 消息流。
6. SSE、WebSocket 和连接状态。
7. 测试结果、Mock 和覆盖率。
8. 内存、GC 和性能曲线。
9. 终端日志与错误堆栈。
10. 阶段项目和综合项目的应用结果预览。

## 13. 验证策略

### 内容测试

- 验证课程规格 Schema、唯一 ID、正确答案、错误反馈和必填字段。
- 验证阶段顺序和先修关系构成无环图。
- 验证每阶段恰好包含 8 个必修知识点和 1 个项目。
- 验证 80 个知识点均有来源、版本和至少 3 道题。
- 在课程声明的 Node.js 版本下运行示例，核对期望日志或结构化结果。

### 逻辑测试

- 覆盖答题、重试、提示、运行、取消、总结和复习入队状态。
- 覆盖进度恢复、阶段解锁、项目解锁和最终项目解锁。
- 覆盖不同执行来源到统一可视化事件的映射。

### 集成测试

- 验证沙箱禁止外部网络、看不到宿主密钥且遵守资源上限。
- 验证可信模板、受控补丁、SSE 事件、取消、超时和清理。
- 验证沙箱故障时的模拟轨迹回退和明确标记。

### 端到端与视觉检查

- 在桌面和移动端完成一个普通知识点、一个阶段项目及最终项目解锁流程。
- 验证刷新后进度恢复、错误反馈可访问、键盘可操作和日志可阅读。
- 对 10 类可视化组件保留稳定截图或视觉回归基线。

## 14. 交付顺序

1. 建立课程 Schema、内容目录、来源治理、进度仓储和现有 4 个案例迁移。
2. 完成阶段 01–03、基础题库和模块/事件循环可视化。
3. 完成阶段 04–06、文件/Stream/HTTP 可视化和首个真实沙箱项目。
4. 完成阶段 07–10、实时/并发/测试/诊断可视化和其余阶段项目。
5. 完成最终综合项目、全课程覆盖检查、移动端和安全验收。

详细文件级任务、测试先后顺序和每次提交边界将在实现计划中定义。

## 15. 完成标准

- 10 个阶段、80 个必修知识点、至少 240 道题、10 个阶段项目和 1 个综合项目全部可导航。
- Node.js 官方 Learn 的实用必修主题全部进入覆盖矩阵，且每课可追溯来源和适用版本。
- 每个知识点具备独立错误反馈、正确答案解释、视觉结果、终端日志和知识总结。
- 普通题能够稳定播放预设轨迹，项目题能够在受限沙箱真实运行并正确降级。
- 学习者能够保存和恢复进度，按规则解锁阶段项目和最终项目。
- 内容、逻辑、集成、端到端和视觉检查全部通过。
