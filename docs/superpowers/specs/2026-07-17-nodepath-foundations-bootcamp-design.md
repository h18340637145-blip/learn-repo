# NodePath「00 基础训练营」设计规格

## 背景

NodePath 当前课程主线覆盖 Node.js 运行时、模块、异步、HTTP、API、并发、实时通信、测试安全和生产诊断。原产品定位默认学习者已经掌握 JavaScript 基础语法，但新的产品目标要求让学习者也能在系统内补齐 Node.js 入门基础、基本语法和基础题库。

因此，本规格新增一个位于正式 10 阶段之前的预备阶段：`00 基础训练营`。它负责把学习者从“会一点 JavaScript 或刚接触 Node.js”带到“可以进入 01 运行时与命令行”的状态。

## 目标

- 为 Node.js 初学者提供系统的基础语法和运行概念学习入口。
- 扩充基础学习题库，让学习者能通过预测输出和概念辨析建立信心。
- 保持现有沉浸式学习模式：讲解、代码、答题、运行可视化、终端日志、总结。
- 不破坏当前 01–10 阶段主线结构，不降低现有课程质量。

## 非目标

- 不引入真实代码沙箱或任意 Node.js 执行能力。
- 不做自由输入编程题。
- 不改变当前本地进度存储模型。
- 不强制学习者完成基础训练营后才能进入正式阶段。

## 课程结构

新增阶段：

```text
00 基础训练营
```

该阶段包含 8 个基础知识点和 1 个阶段项目：

1. Node.js 与 JavaScript 的关系。
2. 变量、类型与 `typeof`。
3. 字符串、数组与对象。
4. 函数、参数与返回值。
5. 条件判断与循环。
6. 错误处理：`try/catch`。
7. `console` 与调试输出。
8. `process`、路径与文件读取入门。
9. 阶段项目：命令行文件统计器。

每个知识点仍然是一个 `LessonSpec`，阶段项目使用 `kind: "stage-project"`。

## 学习体验

基础训练营沿用当前标准流程：

```text
Concept -> Code Case -> Prediction -> Runtime Visualizer -> Terminal Logs -> Summary
```

基础课程的讲解语言应更轻、更直接，减少高级术语。每课需要强调一个可记忆规则，例如：

- Node.js 运行 JavaScript，但额外提供文件、进程、路径等宿主能力。
- `typeof` 看的是运行时值的类型。
- 函数只有被调用时才进入调用栈。
- `try/catch` 只能捕获当前同步执行链或 `await` 恢复点上的异常。

## 题库设计

第一版每个知识点至少包含 1 道可交互题，题目类型优先为：

- 预测输出：给一段短代码，判断终端会打印什么。
- 概念辨析：判断哪个说法最准确。

每个选项必须包含定向反馈，解释为什么对或为什么错。错误反馈不应只是“错误”，而要指出学习者误判的心智模型。

后续可以在不改课程结构的前提下扩展为多题轮转；第一版保持现有 UI 只读取 `questions[0]` 的模式，避免扩大前端状态复杂度。

## 可视化设计

基础训练营的运行可视化不追求复杂场景，重点让学习者看懂“代码如何一步步发生”：

- 变量与类型：数据块进入命名槽位。
- 字符串、数组、对象：集合节点展开字段或索引。
- 函数：参数进入函数调用栈，返回值流回调用点。
- 条件与循环：分支路径亮起，循环轨道重复推进。
- 错误处理：异常从出错点进入 `catch` 边界。
- 文件读取：路径、文件、文本内容、终端输出形成数据流。

这些可视化通过现有 `execution.frames` 和 `execution.visualizer` 表达，不新增可视化基础设施。

## 数据与架构改动

新增文件：

```text
content/lessons/stage-00-foundations.ts
```

修改文件：

```text
content/curriculum.ts
content/lesson-registry.ts
lib/curriculum/validate.ts
tests/curriculum/*
docs/PRODUCT.md
docs/ARTICHECTURE.md
session-handoff.md
```

关键规则：

- 课程目录从 `01–10` 扩展为 `00–10`。
- `00` 阶段是基础阶段，不替代现有正式阶段。
- 校验逻辑需要支持 11 个阶段，其中 `00` 使用特殊阶段编号或稳定 `stageId`。
- 已发布案例数量从 83 增加到 92。
- 左侧导航和阶段星图应自然展示 `00 基础训练营`，但不强制锁住后续课程。

## 兼容性

现有进度数据继续可用。新增课程会产生新的 `lesson.id` 和 `project.id`，不会影响已有 `completedLessonIds`、`completedProjectIds`。

如果学习者已有正式阶段进度，基础训练营默认显示为未完成，不会清除或迁移旧数据。

## 测试策略

至少增加或更新以下测试：

- 课程目录包含 `00` 基础训练营和原有 10 个正式阶段。
- 基础训练营包含 8 个知识点和 1 个阶段项目。
- 注册表可以按 ID 查询全部基础课程。
- 所有基础课程通过 `validateLessonSpec`。
- 已发布案例数量更新为 92。
- 侧边栏和阶段空间能渲染基础阶段。
- 文档中的产品定位不再假设学习者已经掌握 JavaScript 基础。

## 交付标准

- `npm run validate:curriculum` 通过。
- `npm test` 通过。
- `npm run lint` 通过。
- `npm run build` 通过。
- `git diff --check` 通过。
- 浏览器中可以从左侧进入 `00 基础训练营` 并完成至少第一课。

## 规格自检

- 无占位符：本规格不包含待定章节。
- 无范围冲突：第一版只增加基础课程数据、目录和校验，不引入沙箱或自由输入题。
- 无架构矛盾：继续使用 `LessonSpec`、authored trace 和现有可视化系统。
- 可实现范围明确：一个实现计划可以覆盖课程目录、课程数据、测试、文档和浏览器验收。
