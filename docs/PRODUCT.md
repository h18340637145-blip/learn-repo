# NodePath Product Harness

## Product Summary

NodePath 是一个可视化编程学习平台，当前支持 Node.js 和 Next.js 两条学习路径。它用“先预测、再运行、最后总结”的方式帮助学习者建立技术栈心智模型，而不是只阅读概念说明。

首屏现在是可操作的课程选择入口，学习工作台分别位于 `/nodejs` 和 `/nextjs`。

## Target Learner

主要学习者包括 JavaScript 基础较弱、刚接触 Node.js / Next.js 的新手，也包括已经掌握 JavaScript 基础语法、希望系统理解真实工程行为的学习者。Node.js 路线从基础训练营开始，逐步进入模块、包、异步队列、文件、流、HTTP、进程、实时通信、测试、安全和生产诊断；Next.js 路线从 App Router 基础开始，逐步进入路由、渲染、数据获取、优化、API、认证、数据库、测试部署和高级模式。

体验优化目标：

- 小知识点。
- 具体代码。
- 先预测再解释。
- 可视化运行过程。
- 终端日志反馈。
- 错误选项提供定向反馈。
- 阶段项目复用前面知识。

## Learning Flow

每个知识点遵循：

```text
Concept -> Code Case -> Prediction -> Correct Answer -> Runtime Visualizer -> Summary
```

每个阶段项目遵循同一交互模式，但会组合前面知识：

```text
Project Brief -> Incomplete Code -> Choice -> Simulated Run -> Visual Result -> Knowledge Summary
```

Node.js 全部阶段项目完成后，学习者应进入中大型最终项目：实时协作任务平台，覆盖 API、数据流、实时通信、鉴权、测试、诊断和部署等主题。Next.js 路线的最终项目目标是实时仪表盘，覆盖路由、渲染、数据获取、认证、数据库、部署和高级模式。

### 3D 空间化学习体验

NodePath 采用阶段星图组织每条学习路径。左侧导航展示阶段入口，并展开当前阶段的可点击知识点；主区域继续用阶段星图呈现该阶段知识点和阶段项目。答对题目后，系统根据课程的 `visualizer` 配置展示 3D 运行舱或轻量 fallback，帮助学习者把请求、进程、连接、测试、诊断、路由、渲染和数据流过程记成空间关系。

当前运行舱增强为“知识环绕”视觉：核心执行节点保留在中轴，知识标签围绕节点轨道漂浮，粒子点云随运行状态变亮。项目依赖声明包含 `three.quarks`、`three-nebula` 和 `proton-engine`，当前稳定版本优先使用 Three.js / React Three Fiber 渲染可控粒子层，并保留专业粒子引擎桥接点。

运行舱会根据容器尺寸和浏览器媒体查询动态选择 3D Canvas 或 fallback。桌面宽布局优先展示 3D 空间场景；移动端、减少动态效果、WebGL 不可用或运行舱宽度不足时，使用深色 fallback 保留运行顺序和终端语义，确保浏览器缩放后不出现空白区域。

### 游戏化沉浸反馈

参考成就宇宙类界面，NodePath 将课程完成包装成“知识芯片解锁”，阶段项目完成包装成“阶段徽章铸造”。当前版本只实现纯前端反馈：Mission HUD、鼠标火花、HUD 扫描线、成就解锁弹层、学习卡片 3D hover 和星图节点能量环。它们不改变课程解锁规则，也不引入后端成就系统。

重点阶段当前映射到主题化运行场景：

- HTTP 基础：请求响应管线。
- API 与服务设计：服务边界和可靠接口。
- 进程与并发：主线程、线程池、Worker 和 IPC。
- 实时通信：连接星网、心跳和广播。
- 测试与安全：断言光束和安全边界。
- 诊断与生产工程：CPU、Heap、GC 和发布检查。

## Runtime Model

普通课程当前使用 authored trace：课程内容提供可信代码样例、预设运行帧和日志，浏览器只播放这些确定性帧。

真实 Node.js 编译或沙箱执行不在当前阶段。后续项目沙箱必须作为独立计划设计，不能在 Next.js 进程里运行学习者提交的任意代码。

## Progress

当前进度保存在浏览器 `localStorage` 中：

- 完成知识点后记录 `completedLessonIds`。
- 完成阶段项目后记录 `completedProjectIds`。
- 每条学习路径拥有独立进度快照，并在快照中记录 `courseId`。
- 刷新页面后会恢复进度。
- 清理站点数据会清除进度。

Supabase 登录、跨设备同步和严格跨阶段解锁属于后续计划。

## Product Principles

- 让学习者始终处在“做”的循环里。
- 用视觉因果关系替代长篇概念堆叠。
- 错误答案要有帮助，而不是惩罚。
- 每课都应该留下一个可记忆规则。
- 阶段项目必须显式复用前面概念。
- UI 应该适合反复学习：信息密度足够，但视觉保持克制。
- 酷炫效果必须服务学习，不遮挡代码和题目。
- 关键时刻可以强反馈，常态阅读保持稳定。

## Success Criteria

短期：

- 课程目录可校验，已发布案例可完整运行。
- 错误选项能给出当前题目的定向反馈。
- 完成进度能在刷新后恢复。
- `npm run validate:curriculum`、`npm test`、`npm run lint`、`npm run build` 可通过。

中期：

- 补齐阶段 04 剩余知识点。
- 题库覆盖 prediction、diagnosis、transfer 三类问题。
- 增加更多可视化类型，尤其是模块图、事件循环、Stream/HTTP 流程和并发图。

长期：

- 安全的项目沙箱。
- 更丰富的项目输出，包括日志、图表和小型渲染结果。
- 最终项目覆盖大多数 Node.js 学习知识点。
